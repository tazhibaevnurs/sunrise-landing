# Sunrise Family — бэкенд (Django)

## Установка и запуск

```bash
# Виртуальное окружение уже создано в .venv
.\.venv\Scripts\activate   # Windows
pip install -r requirements.txt
python manage.py migrate
python manage.py createsuperuser   # для входа в админку
python manage.py runserver
```

- Сайт: http://127.0.0.1:8000/
- Админка: http://127.0.0.1:8000/admin/

## Модели и админка

### 1. Заявки (Application)
- **Модель:** тип заявки (донорство / суррогатное материнство), имя, телефон, возраст (для донорства), город (для суррогатного материнства), дата, флаг «Обработана».
- **Формы на сайте:** блок «Узнать о донорстве» (имя, телефон, возраст) и блок «Стать частью чуда» (имя, телефон, город). Отправка через POST с CSRF, сохранение в БД и редирект с сообщением.
- **В админке:** список заявок, фильтры по типу и дате, поиск, отметка «Обработана» прямо в списке.

### 2. Изображения сайта (SiteImage)
- **Модель:** код (slug): hero, about, donation, surrogacy, story_1, story_2; файл изображения и/или внешняя ссылка (url_override); подпись (alt).
- Если для slug есть запись с загруженным файлом или `url_override` — на сайте подставляется оно; иначе используются дефолтные URL из шаблона.
- **В админке:** добавление/редактирование записей по каждому slug, загрузка картинки или ввод ссылки, подпись. Поле slug при редактировании недоступно для изменения (только при создании).

### 3. FAQ (FAQItem)
- **Модель:** вопрос, ответ, порядок вывода.
- На сайте блок FAQ строится из записей по полю `order`. Если записей нет — показывается встроенный в шаблон дефолтный текст.
- **В админке:** CRUD вопросов, изменение порядка (поле `order`).

## Структура проекта

- `sunrise_site/` — настройки Django (settings, urls, wsgi).
- `main/` — приложение: models (Application, SiteImage, FAQItem), forms, views, urls, admin.
- `templates/main/home.html` — главная страница (лендинг с формами, динамическими фото и FAQ).
- Медиафайлы сохраняются в `media/site_images/` (см. MEDIA_ROOT в settings).

## Запуск без виртуального окружения

Если используете свой интерпретатор Python:

```bash
pip install Django Pillow
python manage.py migrate
python manage.py createsuperuser
python manage.py runserver
```
